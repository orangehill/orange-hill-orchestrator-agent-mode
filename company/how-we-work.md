# How we work (AI-native)

This is the operating model — the same one whether you join in delivery or in sales. If it sounds like how you already want to work, we'll get along.

## The core loop
1. **A human chooses a frame** worth working inside — the goal, the standard, what "good" means.
2. **Agents do the execution** — the building, the drafting, the grunt work, at volume.
3. **A human reviews** for accuracy, taste, risk, and fit to the live situation.
4. **The learning is saved** — as a spec, a prompt, a checklist, a repo rule — so the next run is better. It compounds.

That's the practical meaning of our whole worldview: when AI makes execution cheap, the valuable human work moves to **framing, judgment, and review**.

## What that looks like day to day
- **Agents build; humans judge.** We describe and verify; we don't type code line-by-line. The craft is being a product architect, not a typist.
- **Many parallel threads, one serial mind.** You might have several projects running at once with agents executing — but judgment is serial, so we protect it: load → incubate → commit, with deliberate thinking time. (See *Judgment Doesn't Parallelize* in `orange-hill-dna.md`.)
- **Voice-first and always-on.** We lean on coordinator agents that mobilize other agents and report back in plain language. Work happens from a laptop, a phone, a voice note.
- **Spec/PRD-driven delivery.** Clear framing up front; agents fill in the execution; we verify against the spec.
- **Craft on the visible *and* invisible parts.** Beautiful artifacts, good UX, the details no one will notice. Someone gave a damn.
- **Teach, don't gatekeep.** With clients, capability transfer over dependency. Internally, save what you learn so the team compounds.

## What we don't do
- Hand-code things an agent should build.
- Ship polished emptiness ("workslop") to hit a deadline.
- Chase every new tool and benchmark. We watch bottlenecks, not benchmarks.
- Wire things together quickly and hope. We build systems that hold.
